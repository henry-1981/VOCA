# BrideVOCA v0.2 Stabilization & Deploy — Design

**Date:** 2026-03-13
**Spec:** `2026-03-13-v0.2-stabilize-deploy-spec.md`

## Design Direction

기존 마법 아카데미 테마를 유지하면서, 전 화면의 시각적 완성도를 높이고
게이미피케이션 요소를 추가하여 아이들의 학습 동기를 강화한다.

## Screen-by-Screen Design

### Main Hub

**현재:** 아카데미 외관 CSS 렌더링 (탑, 성벽, 별, 게이트 완성)
**추가 작업:**

1. **시간대별 배경 변화** (#8)
   - dawn (5-7시): 핑크-오렌지 그라데이션, 별 희미
   - morning (7-12시): 밝은 하늘, 금빛 햇살
   - afternoon (12-17시): 푸른 하늘, 구름
   - evening (17-20시): 현재 구현 (보라-주황 석양)
   - night (20-5시): 짙은 남색, 별 강조, 달

2. **프로필 전환 UI** (#40)
   - Hub 상단 헤더 영역에 현재 아이 이름 + 전환 버튼
   - 전환 시 아바타, 이름, 진행 데이터 모두 교체
   - 디바이스 바인딩 갱신 (localStorage + Firestore)
   - 전환 확인 다이얼로그 (실수 방지)

3. **진행률/성취 시각화 강화** (#26)
   - Today 카드에 현재 Day 진행 상태 표시 (Learn/Test 완료 여부)
   - Character 카드에 현재 레벨/스트릭 미니 배지

### Today Stage

**현재:** Learn→Test 스테이지 전환 구현 완료
**추가 작업:**

1. **Learn→Test 전환 축하 연출** (#16)
   - Learn 완료 시 "문이 열리는" 애니메이션 (0.8s)
   - Test 카드가 glow와 함께 확대되며 primary로 전환
   - 짧은 성취 메시지: "학습 완료! 이제 테스트에 도전하세요"

2. **Day 선택 UI** (#3)
   - Today 화면 상단에 "다른 Day 선택" 버튼
   - 클릭 시 전체 20 Day 목록 모달 표시
   - 각 Day: 번호, 제목, 완료/미완료 상태, checkpoint 표시
   - 완료된 Day: 체크마크 + 점수 미니뱃지
   - 미완료 Day: 접근 가능 (spec: 미래 Day 조기 완료 허용)

### Learn

**현재:** 영단어 중심, 오디오 재생, 이전/다음 카드
**추가 작업:**

1. **예문 오디오 버튼 icon-only** (#14)
   - "예문 듣기" 텍스트 → ▶ 아이콘 버튼으로 변경
   - 크기: 32x32, 터치 영역: 44x44 (iPad 터치 가이드)

### Test (Learning Test)

**현재:** 2x2 4지선다, EN↔KO 혼합, 피드백 구현
**추가 작업:**

1. **정답 이펙트** (#18)
   - 선택지 초록 하이라이트 + 짧은 스파클 (300ms)
   - 화면 상단에서 작은 별 파티클 3-5개 떨어짐
   - 밝은 플래시 (opacity 0→0.15→0, 200ms)

2. **오답 이펙트** (#19)
   - 선택지 붉은 하이라이트 + 부드러운 흔들림 (400ms)
   - "다시 해보자" 톤의 메시지 (현재 구현 유지)
   - 과도한 실패 연출 금지 (spec: 절제 원칙)

3. **콤보 카운터** (#20)
   - 연속 정답 시 카운터 표시: "x2", "x3", "x5"...
   - 3연속부터 카운터 크기/색상 강조
   - 5연속 시 보너스 XP (+10 추가)
   - 콤보 끊겨도 부정적 연출 없음 (조용히 리셋)

4. **+XP 플로팅 숫자** (#23)
   - 정답 시 "+5 XP" 텍스트가 위로 떠오르며 fade out (600ms)
   - 콤보 보너스 시 "+15 XP" 더 크게

### Checkpoint Test

**현재:** 기본 스타일링, LearningTestScreen과 불일치
**추가 작업:**

1. **스타일링 통일** (#15)
   - LearningTestScreen과 동일한 그라데이션 배경
   - 동일한 진행 바, 선택지 스타일
   - 게이미피케이션 이펙트도 동일 적용

### Review

**현재:** 차분한 복습실 톤 구현, 부분적 moonlight 테마
**추가 작업:**

1. **Review Landing silver 악센트** (#13)
   - 텍스트 색상: `text-slate-300` → `text-silver` 계열
   - 테두리: `border-sky-300/12` → `border-slate-200/20` (은빛)
   - CTA 버튼: 기존 파란색 유지하되 은색 glow 추가

2. **Review Session moonlight 강화** (#12)
   - 배경: 현재 파란 그라데이션에 은빛 달빛 레이어 추가
   - 선택지: 은색 테두리, 부드러운 hover glow
   - 피드백: 파스텔 톤으로 절제 (정답: 연한 민트, 오답: 연한 라벤더)
   - 게이미피케이션 이펙트 **최소** 적용 (절제 원칙: Review는 가장 차분하게)

### History

**현재:** 따뜻한 크림/탄 도서관 느낌 구현
**추가 작업:**

1. **Gold/sepia 악센트 보강** (#17)
   - 카드 테두리: sepia 톤 (`border-amber-700/20`)
   - Day 제목: gold 강조 (`text-amber-800`)
   - 배경 텍스처: 미세한 종이 질감 오버레이

### Character

**현재:** 다크 배경, 텍스트 placeholder 아바타, 스탯 패널
**추가 작업:**

1. **아바타 이미지 렌더링** (#9)
   - `{childName}` 텍스트 → `<Image src={avatarSrc}>` 교체
   - 아바타 주변 마법 오라 glow

2. **마법 연구실 장식** (#10)
   - 배경에 선반/책 실루엣 (CSS)
   - 떠다니는 작은 스파클 파티클 (4-6개, animate-float)
   - 은은한 마법 오라 (radial-gradient)

3. **스탯 패널 floating magic** (#11)
   - Level/Streak 패널에 미세 glow pulse 애니메이션
   - 호버 시 패널 살짝 떠오르는 효과 (translateY -2px)

## Gamification System Design

### 이펙트 절제 원칙 (#38)

| 화면 | 이펙트 수준 | 근거 |
|---|---|---|
| Learn | 없음 | 집중 학습, 자극 최소화 |
| Test | 중간 | 도전 느낌, 짧고 명확한 피드백 |
| Review | 최소 | 차분한 회복, 은은한 색상만 |
| Day 완료 | 강함 | 성취 순간, 큰 보상감 |
| 레벨업 | 강함 | 마일스톤, 기억에 남는 연출 |
| Hub | 약함 | 정보 전달 중심, 과도한 움직임 지양 |

### 이펙트 시간 가이드

- 정답/오답 피드백: **300-400ms**
- 콤보 표시: **500ms** (fade in → hold → fade out)
- +XP 플로팅: **600ms**
- Day 완료 축하: **1.5-2s** (풀 애니메이션)
- 레벨업 연출: **2-3s** (풀스크린)
- Streak 강조: **800ms**

### Day 완료 축하 (#22)

- 화면 중앙에 "Day XX 완료!" 텍스트
- 폭죽 파티클 (좌우에서 분사, 1.5s)
- 획득 XP 총합 표시
- "다음 Day로" 또는 "Hub로 돌아가기" CTA

### 레벨업 연출 (#24)

- 풀스크린 오버레이 (반투명 다크)
- 중앙에 "Level Up!" + 새 레벨 번호
- 금빛 원형 확산 이펙트
- 아바타 이미지 표시 (성장 느낌)
- 2-3초 후 자동 닫힘 또는 탭으로 닫기

### Streak 강조 (#25)

- Hub 진입 시 streak 유지 중이면 배지 glow
- streak 갱신 시 "+1일 연속!" 토스트 (800ms)
- streak 끊김 시: 조용히 리셋, 부정적 연출 없음

## 오프라인 대응 (#37)

### 전략

- Firestore `enablePersistence()` 활성화
- 학습 데이터(Day 콘텐츠)는 첫 로드 시 로컬 캐시
- 오프라인 상태에서도 Learn, Test 진행 가능
- 재접속 시 Firestore가 자동 동기화
- 오프라인 인디케이터: Hub 상단에 "오프라인" 배지 (눈에 띄지 않게)

### 제한

- 프로비저닝은 온라인 필수 (Firebase Auth)
- 프로필 전환은 온라인 필수 (Firestore 조회)

## AI 에셋 품질 기준 (#39)

### UI/UX 이미지 (Gemini GCloud)

- 용도: 아바타, 배경, 아이콘, 장식 에셋
- 포맷: WebP (투명 배경 필요 시 PNG)
- 최대 사이즈: 512x512 (아바타), 1920x1080 (배경)
- 스타일 가이드: 마법 아카데미 테마, 따뜻한 판타지, 아이 친화적
- 품질 검수: 생성 후 수동 확인, 부적절한 이미지 재생성

### 효과음 (AI 생성)

- 포맷: MP3
- 볼륨 정규화: -14 LUFS 기준
- 길이: 최대 2초
- 품질 검수: 생성 후 청취 확인

## Hub 프로필 전환 (#40)

### 동작

1. Hub 상단에 현재 아이 이름 + 아바타 미니 아이콘
2. 탭 시 전환 확인 다이얼로그: "{otherChild}(으)로 전환할까요?"
3. 확인 시:
   - 로컬 디바이스 바인딩 갱신
   - Firestore 디바이스 레지스트리 업데이트
   - 새 childId로 앱 컨텍스트 리로드
4. Hub가 새 아이의 데이터로 갱신

### 제약

- 온라인 상태에서만 전환 가능
- 전환 후 이전 아이의 미저장 데이터는 Firestore에 먼저 flush
